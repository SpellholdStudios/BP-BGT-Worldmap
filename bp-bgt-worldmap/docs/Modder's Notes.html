Overview

BP-BGT-Worldmap Modder's Notes is intended for modders who wish to build modifications with worldmap icons.

A special thanks goes to K4thos for the two functions ADD_WORLDMAP and ADD_MAP_ICON_EE (Even if I improved them a bit :) ) This thanks also goes to Mike1072 for his ADD_MAP_ICON function. I hope you don't mind that I included it in the world map.

Adding new icons to BP-BGT-Worldmap

General

The following is just an example how to add areas with icons. In the meantime, several variants have developed. Some easier, some more difficult, some more complete.

In general, it is possible to add new map icons during the installation and assign them to the corresponding areas.

Every mod that should be compatible with the BP-BGT-Worldmap needs a directory "worldmap" in the main MOD_FOLDER.

The following files must be contained there:

  1. areas.tbl
  2. links.tbl
  3. worldmap.tra

How the areas.tbl, links.tbl and worldmap.tra are built, have a look at example mods.

Two cases must be distinguished. Can I install my mod after the BP-BGT worldmap, or must it be before the BP-BGT worldmap. Depending on this, the installation differs.

Mods after BP-BGT-Worldmap

Here comes a code example for the installation.

First we need to append the icons to the BPBGTMAP.BAM

EE engine variant

ACTION_IF GAME_IS ~bg2ee eet~ BEGIN
  INCLUDE ~%MOD_FOLDER%/lib/add_map_icons_ee.tpa~
  LAF ADD_MAP_ICONS_EE STR_VAR path_to_icons = EVAL ~%MOD_FOLDER%/Worldmap/ee/icon.bam~ patch_to_pvrz = EVAL ~%MOD_FOLDER%/Worldmap/ee~ RET new_icon_index icons_added END
END

original engine variant

ACTION_IF GAME_IS ~tob bgt~ BEGIN //bg2, bgt
  INCLUDE ~%MOD_FOLDER%/lib/add_map_icons.tpa~
  LAF ADD_MAP_ICONS STR_VAR path_to_icons = EVAL ~%MOD_FOLDER%/Worldmap/bg/icon.bam~ RET new_icon_index icons_added END
END

ADD_MAP_ICONS_EE and ADD_MAP_ICONS are functions that appends bam icons to the BPBGTMAP.BAM, depending on engine.

Then we need to create the remapped_icons array. The new icons have different numbers in the BPBGTMAP.BAM file.

ACTION_DEFINE_ASSOCIATIVE_ARRAY remapped_icons BEGIN 10 => ~%new_icon_index%~ END

When adding multiple icons, it becomes difficult to add the numbers to the array manually. Therefore we can also have the array created automatically with the following function CREATE_REMAPPED_ICONS.

//EE engine
ACTION_IF GAME_IS ~bg2ee eet~ BEGIN
  INCLUDE ~%MOD_FOLDER%/lib/add_map_icons_ee.tpa~
  LAF ADD_MAP_ICONS_EE STR_VAR path_to_icons = EVAL ~%MOD_FOLDER%/Worldmap/ee/icon.bam~ patch_to_pvrz = EVAL ~%MOD_FOLDER%/Worldmap/ee~ RET new_icon_index icons_added END
  // erstellt automatisch aus deiner area.tbl Datei den remapped_icons array
  ACTION_IF %icons_added% > 0 BEGIN
	LAF CREATE_REMAPPED_ICONS INT_VAR verbose = 1 STR_VAR path_to_areas = EVAL ~%MOD_FOLDER%/Worldmap/areas_eet.tbl~ RET_ARRAY remapped_icons END
  END
END
//original engine
ACTION_IF GAME_IS ~tob bgt~ BEGIN //bg2, bgt
  INCLUDE ~%MOD_FOLDER%/lib/add_map_icons.tpa~
  LAF ADD_MAP_ICONS STR_VAR path_to_icons = EVAL ~%MOD_FOLDER%/Worldmap/bg/icon.bam~ RET new_icon_index icons_added END
  // erstellt automatisch aus deiner area.tbl Datei den remapped_icons array
  ACTION_IF %icons_added% > 0 BEGIN
	LAF CREATE_REMAPPED_ICONS INT_VAR verbose = 1 STR_VAR path_to_areas = EVAL ~%MOD_FOLDER%/Worldmap/areas_bgt.tbl~ RET_ARRAY remapped_icons END
  END
END

Now we use the actual function to install the new areas.

//Worldmap for EET
ACTION_IF GAME_IS ~eet~ BEGIN
	INCLUDE ~%MOD_FOLDER%/lib/add_worldmap_tbl.tpa~
	LAF ADD_WORLDMAP_TBL
		INT_VAR
		verbose = 0
		inclSv = 0
		patch_bp_files = 1
		patch_mastarea = 1
		STR_VAR
		path_to_areas = EVAL ~%MOD_FOLDER%/Worldmap/areas_eet.tbl~
		path_to_areas_bp = EVAL ~%MOD_FOLDER%/Worldmap/areas_bp_bgt.tbl~
		path_to_links = EVAL ~%MOD_FOLDER%/Worldmap/links.tbl~
		path_to_links_bp = EVAL ~%MOD_FOLDER%/Worldmap/links_bp_bgt.tbl~
		path_to_trans = EVAL ~%MOD_FOLDER%/tra/autotra/%LANGUAGE%/worldmap.tra~
		icons_table = ~remapped_icons~
		worldmap = EVAL ~%worldmap%~
	END
END

This function installs the new area to the existing worldmap. path_to_areas_bp means that it is for the BP-BGT worldmap and path_to_areas is for the normal one in the game. Here you can of course also query the different games (tob, bgt, bgee, bg2ee ...). It also does not matter if you have the bp variant with you and do not want to install BP-BGT-Worldmap, then the function does not use the areas_bp.tbl.

You can use for example this code for a bg2ee or eet installation of your mod with and without BP-BGT-Worldmap.

Here you can find a list of all function variables and their meanings:

DEFINE_ACTION_FUNCTION ~ADD_WORLDMAP_TBL~
	INT_VAR
	inclSv = 0            // to 1 if you want to patch saved game worldmaps as well as the master worldmap. NB: changes to saved games are uninstallable
	verbose = 0           // to 1 to enable debug messages
	patch_bp_files = 0    // to 1 to patch/create gameFolder/Worldmap files used by BP-BGT Worldmap for wmp creation (needs path_to_areas_bp, path_to_links, path_to_trans)
	patch_mastarea = 0    // to 1 to patch MASTAREA.2da if area entries doesn't exist there yet
	add_x = 0             // add this value to area's X (east-west) coordinate specified in TBL file (BP-BGT Worldmap areas file is excluded)
	add_y = 0             // add this value to area's Y (north-south) coordinate specified in TBL file (BP-BGT Worldmap areas file is excluded)
	STR_VAR
	path_to_areas = ~~    // full path to the text file containing your areas data, e.g. ~mymod/tbl/areas.tbl~
	path_to_areas_bp = ~~ // as above but this points to file used by BP-BGT Worldmap - this way if some save is detected to have BP-BGT Worldmap wmp stored it will use different coordinates / values
	path_to_links = ~~    // full path to the text file containing your links data, e.g. ~mymod/tbl/links.tbl~
	path_to_links_bp = ~~ // as above but this points to file used by BP-BGT Worldmap
	path_to_trans = ~~    // full path to the text file containing your translation data, e.g. ~mymod/language/%LANGUAGE%/worldmap.tra~
	strref_array = ~~     // array containing pre-set strrefs (ARE name => strref) for area names and tooltips
	worldmap = ~worldmap~ // which .wmp file should be patched, e.g. ~worldm25~
	icons_table = ~~      // array containing remapped icons (old_index => new_index) for path_to_areas (use alongside function that can patch BAMv1/BAMv2 icon file)
	RET
	areas_added           // how many areas were successfully added. if this is 0, you have a problem
	links_added           // how many links were successfully added. if this is 0, you have a problem
	//areNum_array          // array containing worldmap entry number for each added area (name => areNum), e.g. number evaluation code: TEXT_SPRINT areNum $areNum_array(~name~)
	//commented out because of missing WeiDU feature: http://gibberlings3.net/forums/index.php?showtopic=28123

All functions can be found in the following directory: bp-bgt-worldmap\modders_notes\code-repositories

Mods before BP-BGT-Worldmap

Unfortunately, the icons cannot be installed before installing the BP-BGT worldmap because the corresponding bam file does not exist. Therefore it is done with the following short code. If icons are needed after all, you have to add them manually to the worldmap BAM file in advance. 

ACTION_IF FILE_EXISTS ~Worldmap/map_mods_areas.tbl~ THEN BEGIN
  COPY ~Worldmap/map_mods_areas.tbl~  ~Worldmap~
   APPEND_FILE ~CtB/Worldmap/areas.tbl~

  //preliminary step - making LANGUAGE temporary file until the new WeiDU
  COPY - ~CtB/Worldmap/%LANGUAGE%/worldmap.tra~  ~tmp_worldmap.tra~
  COPY ~Worldmap/map_mods_trans.tra~  ~Worldmap~
   APPEND_FILE ~tmp_worldmap.tra~

  ACTION_IF FILE_EXISTS ~Worldmap/map_mods_links_patch.tbl~ THEN BEGIN
    COPY ~Worldmap/map_mods_links_patch.tbl~  ~Worldmap~
     APPEND_FILE ~CtB/Worldmap/links_patch.tbl~
  END ELSE BEGIN
    COPY ~CtB/Worldmap/links_patch.tbl~ ~Worldmap/map_mods_links_patch.tbl~
  END
END
ELSE BEGIN
  MKDIR ~Worldmap~
  COPY ~CtB/Worldmap/areas.tbl~ ~Worldmap/map_mods_areas.tbl~
  COPY ~CtB/Worldmap/links.tbl~ ~Worldmap/map_mods_links.tbl~
  COPY ~CtB/Worldmap/%LANGUAGE%/worldmap.tra~ ~Worldmap/map_mods_trans.tra~
END

This is an excerpt from "Check the Bodies" as an example of the installation before the BP-BGT-worldmap.

Generating BAM v1 file for Classic Engine

BAM v1 file that is used in original Baldur's Gate 2 game is running out of colors in the palette. Each insertion of a new icon causes to lose some color information. To keep it close to the original file, there is a workaround to generate BAM v1 file from EE PVRZ-based BAM v2 file.

You should regenerate the file after adding new icon, to preform this action you need to:

  1. Install BP-BGT-Worldmap modification on a EE game (version from you want to generate new BAM)
  2. Open the EE game in NearInfinity
  3. Use "Tools->Mass Export" to batch export and convert all BAM-related PVRZ files to PNG:
    1. Select PVRZ as a type to export
    2. Select some output directory
    3. Use "Resource name filter" to filter list of files to export (for example "MOS10.*" with "Use regular expressions" checkbox checked)
    4. Check "Export MOS/PVRZ/TIS as PNG" option
    5. Hit "Export" button
  4. Use "Tools->Convert->Image to PVRZ" converter to convert PNG files back to PVRZ
    1. Select directory with newly created PNG files (verify what files you need to convert)
    2. Select output directory (can be the same as for PNG files)
    3. Use compression type "DXT1" to get rid of alpha channel!!!
    4. Hit "Start Convertion" button
  5. Copy new PVRZ files to override directory with your EE game.
  6. Use "Game->Refresh tree" to reload BAM file.
  7. Search for and open BPBGTMAP.BAM file
  8. Click "Edit BAM" button
    1. Select all frames (Ctrl+A shortcut) and check "Compress frame"
    2. Select "BAM Version" to "Legacy (v1)"
    3. Click "Save output file" button and save in the repository folder

NEVER SAVE BPBGTMAP.BAM V1 FILE AS PVRZ-BASED BAM V2!

Defining icon positions depending on map size

BP-BGT-Worldmap v13 introduced a slightly different version of the huge map. The northern part of FaerÃ»n has been improved and is no longer compatible with older versions of the worldmap. Thus, if your modification uses Icewind Dale "insert" on a huge map or simply has different positions for large and huge map versions, you can adjust these coordinates by creating a patch file.

During installation, BP-BGT-Worldmap will check for all icons' coordinates to decide if the huge map version is required. To override those settings, you can create a patch file that contains new icons' positions for large or huge versions of worldmap.

In your mod's worldmap subdirectory, you can patch all data by creating patch files depending on the worldmap size. A file format is the same as normal worldmap table definitions. Available patch files are:

  • For the large worldmap version:
    • map_mods_large_areas.tbl for patching areas data.
    • map_mods_large_links.tbl for patching links data (requires map_mods_large_areas.tbl file).
    • map_mods_large_trans.tra for patching translations (requires map_mods_large_areas.tbl file).
  • For the huge worldmap version:
    • map_mods_huge_areas.tbl for patching areas data.
    • map_mods_huge_links.tbl for patching links data (requires map_mods_huge_areas.tbl file).
    • map_mods_huge_trans.tra for patching translations (requires map_mods_huge_areas.tbl file).
